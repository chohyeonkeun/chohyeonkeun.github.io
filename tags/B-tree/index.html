    B-tree를 사용하는 이유

    • 탐색에서 최악의 경우에도 O(log2n)이 걸리는 균형 이진트리도 굉장히 큰 성과이지만,
    • 데이터가 메인 메모리에 저장되어 있거나,
    • 데이터베이스의 경우 하드디스크에 저장되어 있으면,
    • 데이터의 연산보다는 메모리 접근에 엄청난 시간이 소요된다.
    • 이 문제를 해결하기 위해 B-tree 자료구조가 필요하다.
